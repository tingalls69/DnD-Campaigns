# Campaign Design Principles

## The Archive Rule

The repository is the archive. Active files are the DM's desk. Archive files are not required reading during live play.

## Single Source Of Truth

Every current fact should have one home:

- Live-play desk: `03_THE_LAST_LANTERN/AI_SESSION_PACKET.md`.
- Current situation reference: active `CURRENT_STATE.md`.
- Personality and voice: `PROTAGONISTS.md`, `COMPANIONS.md`, or `NPC_DIRECTORY.md`.
- Mechanics: `CHARACTER_SHEETS.md`.
- Relationships: `RELATIONSHIPS.md`.
- Open mysteries: `UNRESOLVED_THREADS.md`.
- Hidden future truth: `DM_SECRETS.md`.
- Historical record: archived running logs.
- Human-readable longform context: `90_ARCHIVE/Last_Lantern/HUMAN_WIKI_THE_LAST_LANTERN.md`.

## Current Vs Future

Never convert future plans into current truth. Never let NPCs know DM secrets unless they have an in-world reason.

## Current Vs Archive

If active files and archive files conflict, active files win unless the user corrects canon.

## Session Update Order

After a session, update:

1. `03_THE_LAST_LANTERN/AI_SESSION_PACKET.md`
2. `03_THE_LAST_LANTERN/CURRENT_STATE.md`
3. `03_THE_LAST_LANTERN/LIVE_COMBAT_QUICK_SHEET.md` and `CHARACTER_SHEETS.md` for mechanics/resources.
4. `03_THE_LAST_LANTERN/COMPANIONS.md` and `NPC_DIRECTORY.md` if cast changed.
5. `03_THE_LAST_LANTERN/RELATIONSHIPS.md`
6. `03_THE_LAST_LANTERN/UNRESOLVED_THREADS.md`
7. `03_THE_LAST_LANTERN/RUNNING_LOG.md`
8. Archive source notes only after active files are current.
